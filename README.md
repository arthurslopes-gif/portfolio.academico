## 📂 Planejamento e Estrutura de Diretórios

Para garantir a clareza e a organização do repositório, o projeto foi dividido nas seguintes seções mínimas:

* `projetos-academicos/`: Espaço dedicado ao armazenamento de códigos, trabalhos e desafios práticos desenvolvidos durante a minha formação.
* `projetos-pessoais/`: Projetos autorais, testes de código e laboratórios criados para aprendizado autônomo e evolução técnica.
* `documentacao/`: Repositório de arquivos de suporte, como relatórios, slides de apresentações e guias relacionados aos projetos.

## 🛠️ Tecnologias e Ferramentas Utilizadas

* **Git:** Utilizado para o versionamento local do código, controle de histórico e gerenciamento de versões (Versão 1.0).
* **GitHub:** Plataforma de hospedagem remota para compartilhamento, colaboração e segurança do código-fonte.
* **GitHub Pages:** Ferramenta de deploy para publicação da página web estática (`index.html`) a partir da branch principal.
* **Markdown:** Linguagem de marcação utilizada para a construção e formatação detalhada desta documentação.

## 📈 Histórico de Versionamento (Boas Práticas)

O fluxo de trabalho seguiu rigorosamente os conceitos de ciclo de vida de arquivos do Git:
1. **Working Directory:** Criação da estrutura local e desenvolvimento do arquivo HTML inicial.
2. **Staging Area:** Preparação e indexação das alterações com o comando `git add`.
3. **Local Repository:** Registro das versões estáveis (Commits) detalhando as melhorias implementadas.
4. **Remote Repository:** Envio dos arquivos para a nuvem através do comando `git push`.
